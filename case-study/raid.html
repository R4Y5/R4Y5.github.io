CASE STUDY 5

RAID

INSTALLATION.

Description

Implement RAID configurations for redundancy and performance in the server environment. Apply security baseline policies and deploy scripts via GPO.

Overview

Install and configure the following RAID levels:

  • RAID 0 — Striped Volume
  • RAID 1 — Mirrored Volume
  • Spanned Volume
  • RAID 5 — Parity Volume

Steps

  • Prepare Virtual Hard Disks
    • Clone Windows Server 2008 used for DHCP activity
    • Go to Settings → Storage → Add Hard Disk
    • Click Create
    • Check "Pre-allocate full size"
    • Set size to 50 MB
    • Click Finish
    • Repeat to create 10 disks total
  • Attach Disks and Initialize
    • Attach all disks in Settings → Storage
    • Start Windows Server 2008
    • Open Server Manager → Storage → Disk Management
    • Initialize disks using MBR
    • Confirm Disk 0 + Disk 1–10 are ready
  • RAID 0 — Striped Volume
    • Right-click Disk 1 → New Striped Volume
    • Add Disk 2
    • Set name: RAID0
    • Enable quick format
    • Finish setup
  • RAID 1 — Mirrored Volume
    • Right-click Disk 3 → New Mirrored Volume
    • Add Disk 4
    • Set name: RAID1
    • Enable quick format
    • Finish setup
  • Spanned Volume
    • Right-click Disk 5 → New Spanned Volume
    • Add Disk 6 and Disk 7
    • Set name: Spanned
    • Enable quick format
    • Finish setup
  • RAID 5 — Parity Volume
    • Right-click Disk 8 → New RAID 5 Volume
    • Add Disk 9 and Disk 10
    • Set name: RAID5
    • Enable quick format
    • Finish setup
  • RAID Installation Complete
    • RAID0: Disk 1–2 (Performance)
    • RAID1: Disk 3–4 (Fault Tolerance)
    • Spanned: Disk 5–7 (Capacity)
    • RAID5: Disk 8–10 (Balanced Performance & Fault Tolerance)
