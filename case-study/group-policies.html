CASE STUDY 8

GROUP

POLICIES.

Description

Implement Group Policy Objects (GPOs) to enforce department-specific desktop environments and apply security restrictions across the domain. This ensures centralized management, consistent user experience, and enhanced protection for both users and administrators.

Prerequisites

  • Active Directory Domain Services configured
  • Windows Server 2008 R2 or later
  • Organizational Units (OUs) for departments

Steps

  • Department-Specific Desktop Wallpaper
    • Create a GPO (e.g., ACADEMIC WALLPAPER)
    • Navigate to: User Configuration → Policies → Administrative Templates → Desktop → Desktop Wallpaper
    • Enable policy and set wallpaper path
    • Apply and repeat for each department
  • Create User and Admin Policies
    • Define two GPOs: Users Policy and Admin Policy
    • Enable restrictions in Users Policy
    • Disable restrictions in Admin Policy
  • Users Policy Settings
    • Prohibit access to Control Panel and PC settings
    • Prevent access to command prompt and registry tools
    • Deny removable storage access
    • Set password length (12 characters) and maximum age (42 days)
    • Configure account lockout (30 minutes, 3 attempts)
    • Remove Start Menu items (Help, Run, My Pictures, My Music, My Network Places)
    • Add Logoff option to Start Menu
    • Remove Task Manager from Ctrl+Alt+Del
    • Set Internet Explorer title bar to domain name
    • Enable auditing for directory service access
  • Admin Policy Settings
    • Disable restrictions applied to Users Policy
    • Ensure administrators retain full system control
