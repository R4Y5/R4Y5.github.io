CASE STUDY 4

FILE

SHARING.

Overview

Design and deploy file shares, NTFS permissions, and access controls for departmental resources. Design file shares and set NTFS and share permissions for groups.

File Services Requirements

  • All departments/sections should have one public folder — read/write; domain users cannot access other department folders
  • A university public folder where all domain users can access — read/write
  • An installer folder containing applications — administrator access only
  • All folders except Computer Engineering should contain only office documents — .ppt, .doc, .docx, .xls, .pptx, .pdf, etc.
  • Information Technology folder should contain all files — except executable files
  • Storage quota for each folder — 1 GB
  • Map the shared folder for each client workstation

Step 1 — Create Folders

  • Create directories from Part 2 — departments/sections
  • Create University folder
  • Create Installer folder
  • Optional: Run a script to automate folder creation

Step 2 — Configure File Services

  • Server Manager → Roles → Add Roles → File Services
  • Click Next → Next
  • Check all options — except Windows Server 2003 File Services
  • DFS Namespaces: Enter Namespace Name — NameSpace1
  • Click Next → Next
  • In Credentials section: Click Select → type domain admin username and password
  • Click Next → Next
  • Select local disk — C Drive
  • Click Next until installation completes
  • Restart system

Step 3 — Quota Management

  • Start → Administrative Tools → File Server Resource Manager
  • Navigate to Quota Management → Quota Templates
  • Create New Quota Template
    • Template Name: 1GB
    • Space Limit: 1 GB
    • Click Add

Step 4 — File Screening Management

  • General File Screen Template
    • Permissions: Read/Write | Users: All
    • Files: .ppt, .doc, .docx, .xls, .pptx, .pdf, etc.
    • File Template Name: General File Screen — Check all except General
  • IT File Screen Template
    • Permissions: Read/Write | Users: Information Technology department
    • File Screen Template: Check all — except Executable Files
  • Steps to Create Templates
    • File Groups → Create File Group with appropriate name
    • File Template → Create File Template with restrictions as specified

Step 5 — Provision Shares

  • Administrative Tools → Server Manager → Roles → File Services → Share and Storage Management
  • Click Create Provision Share → Open Shared Folder Provision Wizard
  • Select folder location → Available Volumes: C Drive
  • Click Next → Next
  • Check SMB (Server Message Block protocol)
  • Click Next → Next
  • In SMB Permission section: Add user groups with their specific permissions
  • Click Next → Next → Next
  • Apply Quota: Select the 1GB template
  • Apply File Screen: Select appropriate template — General or IT
  • Click Next → Next → Create → Close
  • University Share
    • SMB Share Name: University
    • Permissions: Read/Write | Users: All Domain Users
    • File Screen: General
  • Installer Share
    • SMB Share Name: Installer$
    • Permissions: Read/Write | Users: Administrator only
    • File Screen: General
  • Department Shares — for each department/section from Part 2
    • Permissions: Read/Write | Users: Respective department users only
    • File Screen: General — all departments except Computer Engineering
    • File Screen: All Files allowed — Computer Engineering
    • File Screen: IT template — Information Technology

Step 6 — Map Shared Folders for Client Workstations

  • On each Windows XP / Windows 7 workstation
  • Open File Explorer / My Computer
  • Right-click → Map Network Drive
  • Select drive letter — e.g., Z:, Y:
  • Enter folder path — \\ServerName\ShareName
  • Check "Reconnect at logon"
  • Click Finish
  • Verify mapped drives appear in File Explorer
