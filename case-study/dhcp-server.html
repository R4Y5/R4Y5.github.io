CASE STUDY 3

DHCP

SERVER.

Overview

Configure DHCP services, scopes, and reservations to provide network addressing for the domain. Configure network roles and ensure clients receive proper settings.

DHCP Server Requirements

  • IP Block — 172.31.0.0 /16 & 10.0.0.0 /8
  • Router IP address — valid IP address
  • DNS Server IP
    • Same as DHCP server
    • Google — 8.8.8.8, 8.8.4.4
    • OpenDNS — 208.67.222.222, 208.67.220.220
  • IP address lease expires every — 8 hours
  • Do NOT distribute first 30 and last 30 valid IP addresses

Setup Steps

  • Clone Windows Server 2008
  • Clone Windows 7 & XP
  • Select WS2008 → Settings → Network → Change Adapter 1 & 2 to Internal Network
  • Configure WS2008 Network
    • Settings → Network → Rename Adapter 1 as XPFEMALENET
    • Rename Adapter 2 as 7MALENET
  • Go to WSXP settings → Network → Rename Adapter 1 as XPFEMALENET
  • Go to WS7 settings → Network → Rename Adapter 1 as 7MALENET
  • Start WS2008
  • Configure Networking — Control Panel → Network & Internet → Network Connections → Change Adapter Settings → Right click LAN → Properties
  • Set IPv4 Properties
    • Local Area Connection 1 (Female) — IP: 172.31.0.31 | Subnet: 255.255.255.0 | Gateway: 172.31.0.32 | DNS: 127.0.0.1
    • Local Area Connection 2 (Male) — IP: 10.0.0.31 | Subnet: 255.0.0.0 | Gateway: 10.0.0.32 | DNS: 127.0.0.1
  • Add DHCP Server Role — Start → Administrative Tools → Server Manager → Roles → Add Roles → Select DHCP SERVER
  • Validate settings and install DHCP
  • Open DHCP — Start → Administrative Tools → DHCP
  • Create New Scopes
    • Class A Female-XP Scope
      • Start IP: 172.31.0.1 | End IP: 172.31.0.254
      • Subnet Mask: 255.255.255.0
      • Exclusions: 172.31.0.1–30 & 172.31.0.225–254
      • Duration: 8 hours | Gateway: 172.31.0.32 | Domain: tupt.edu.ph
    • Class B Male-7 Scope
      • Start IP: 10.0.0.1 | End IP: 10.0.0.254
      • Subnet Mask: 255.0.0.0
      • Exclusions: 10.0.0.1–30 & 10.0.0.225–254
      • Duration: 8 hours | Gateway: 10.0.0.32 | Domain: tupt.edu.ph
  • Configure DNS Servers for both scopes — remove 127.0.0.1
    • Primary: 8.8.8.8 — Google
    • Secondary: 8.8.4.4 — Google
    • Tertiary: 208.67.222.222 — OpenDNS
    • Quaternary: 208.67.220.220 — OpenDNS
  • Configure Windows XP
    • Start Windows XP VM while keeping WS2008 running
    • Rename computer: START → Right click My Computer → Properties → Computer Name → Change → LASTNAME_CS-XP
    • Join domain: tupt.edu.ph → OK
    • Enter credentials: Username: Administrator | Password: WS2008 admin password
    • Welcome message → OK → Restart
    • Log in as TUPT\administrator and test with another domain user
  • Configure Windows 7
    • Repeat all steps from Windows XP configuration
    • Rename computer as LASTNAME_CS-WIN7
    • Join domain: tupt.edu.ph with same credentials
    • Log in as TUPT\administrator and test with another domain user
  • Connect Windows XP & 7 using DHCP
    • Start WS2008, WXP & W7
    • Check Address Leases: WS2008 → Start → Administrative Tools → DHCP → Expand Domain → Expand IPv4 → Address Leases
    • Class B Male network should show W7 connected
    • Class A Female network should show WXP connected
    • Verify on W7: Command Prompt → ipconfig /all — DHCP enabled with IP from Class B Male
    • Verify on WXP with same command — should show Class A Female IP
  • Troubleshooting: Domain Connection Errors
    • If error "Cannot connect to your domain controller" occurs:
    • Verify DHCP → Class A or B → Scope Options → 006 DNS Servers has NO 127.0.0.1
    • Run in Command Prompt: ipconfig /release → ipconfig /renew → ipconfig /all
    • Verify DNS servers in ipconfig output match scope options
    • Attempt to reconnect to domain
