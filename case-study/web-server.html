CASE STUDY 7

WEB

SERVER.

Description

Deploy and configure web server roles (IIS) to host academic web applications and services. Implement security baselines and plan patch deployment.

Requirements

  • Create a university website for IIS 7
  • Create a website for each department
  • Organize departments as subdirectories in wwwroot

Steps

  • Install Web Server Role
    • Open Server Manager
    • Navigate to Roles
    • Click "Add Role"
    • Select Web Server (IIS)
    • Follow installation prompts
  • Add Web Server Features
    • Enable Application Development
    • Enable IIS 6 Management Compatibility
    • Enable FTP Server
    • Download AnyToISO Converter
    • Install tool for content management
  • Create and Configure Website
    • Open IIS Manager
    • Right-click Sites → Add Website
    • Site Name: tupt
    • Path: C:\inetpub\wwwroot\tupt
    • Create "tupt" folder in wwwroot
    • IP Address: 172.31.0.31
    • Host Name: tupt.edu.ph
    • Click OK
  • Configure Output Caching
    • Select "tupt" site in IIS Manager
    • Open Output Caching
    • Add cache rule
    • Extension: .aspx
    • Enable User Mode Caching
    • Save settings
  • Prepare Website Content
    • Navigate to wwwroot → tupt
    • Copy iisstart.htm as homepage
    • Create folders:
      • Academics
      • Administrative
      • ComputerEngineering
      • InformationTechnology
    • Add HTML / ASP.NET files to each folder
  • Test Website Access
    • Right-click site → Browse
    • Test locally on Windows Server 2008
    • Test on Windows 7: http://tupt.edu.ph/
    • Test on Windows XP: http://tupt.edu.ph/
    • Verify all pages load correctly
  • Web Server Setup Complete
    • Hosting university website
    • Serving department content
    • Caching .aspx pages for performance
    • Accessible across all client machines
